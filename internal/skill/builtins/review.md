Review the current code changes for correctness:
1. Run `git diff` (and `git diff --staged`) to see the changes.
2. Look for correctness bugs, missing error handling, edge cases, and unclear code.
3. Report findings concisely, each with a file:line reference and a suggested fix.
Focus on real issues; do not nitpick style that a formatter would handle.
